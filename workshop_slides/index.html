Cross Site Scripting AKA XSS

Definition:

Cross Site Scripting attacks are type of Injection attacks, where an attacker places malicious code usually javasscript in location where user can see. {Ex: Froms}

Impact
  • Steal Cookies to impersonate a User
  • Load Malicious code into a user session
  • Redirect User to a Malicious Website
  • Cause Denial of Service
Root Cause

Trusting user input, and using user input to modify the application or reflecting the user input in the application

Types of XSS
  • Reflected XSS
  • Stored XSS
  • DOM BASES XSS
Counter Measures
  1. Never trust user input
    • Escape HTML, Javascript data when modifying HTML content of the page
    • Sanitize user input with a standardized library.Do not implement your own library
Counter Measures Contd..
  • Use HTTPOnly Cookie flag: HTTPOnly Cookie denies Scripts or user loaded code to gain access to the session Cookies. This will prevent users from stealing session cookies.
  • Implement Content Security Policy

    Missing Function Level Access Control

    Defnition
    Related to Session management and authentication. Developers and designers of an application sometimes implement faulty mechanisms to implement access control. This leads an attacker to access functionality which the user is restricted from accessing.

    Root Cause

    • Access control checks are not made on all user actions post login.
    • Depending on Client side controls to implement access control mechanisms

    Countermeasures

    • Perform authentication and authorization checks on all user actions.
    • Enforce Server Side checks and do not depend only on Client Side Validation.

    Insecure Direct Object Reference

    Defnition
    Related to Session management and authentication. Developers and designers of an application sometimes implement faulty mechanisms to implement access control. Missing Function Level access control and insecure direct object access are very closely related.
    Missing function level access is about accessing functionality the user doesn't have rights to whe reas Insecure Direct Object Access is more related to Sensitive data exposure.

    Password Cracking Methods

    As the name suggests the attacker tries to guess the username and password of a user.

    Some of the password cracking methods are

    • Manually guessing username and passwords
    • Brute force attack of passwords
    • Dictionary attacks
    • Rainbow Tables
    • Phishing
    • Social Engineering
    Counter Measures
    • Enforce a strong Password policy.Use atleast 8 characters with a mix of Upper case,lower cases numbers and special characters.
    • Use CAPTCHA mechanisms on the login page to prevent automated guessing of usernames and passwords
    • Implement account lockout policy after 3 or 5 invalid login attempts depending on the application

    Software with known vulnerabilites

    Defnition
    Most applications developed today depend on a lot of other components and most of these components are not developed by the same team as the application. For Ex: An application might use a specific Java version or MySQL.
    Vulnerabilites in these components weakens the security posture of an application.

    Countermeasures

    • Keep track of all components and versions in use.
    • Monitor public databases to identify if any components in use are vulnerable and update the packaes as advised.

    CSRF - Cross-site Request Forgery

    CSRF
    CSRF is a web application attack that leverages the trust between the Authenticated users and web applications to force the users to carry on aribitary actions on behalf of an attacker.
    Also known as confused Deputy attack
    Impact

    Perform actions on the user's account without the user's consent

    Root Cause
    • HTTP is a stateless protocol
    • Web Browser's inability to properly distinguish between a legitimate and unauthorized request.
    Countermeasures
    • Double-posted cookie
    • Use Anti-CSRF tokens
    • Re-authentication of the user while carrying out sensitive actions
